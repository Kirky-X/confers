# 提案：添加基础设施要素

## 摘要

为 Confers 项目补充缺失的基础设施要素：基准测试、模糊测试、CI/CD 配置和 ADR 文档，使项目达到生产就绪状态。

## 背景

在分析 `dev-v2.md` 需求文档时，发现项目核心功能已完成 95%+，但基础设施部分完全缺失：
- 基准测试目录为空文件
- 模糊测试目录不存在
- CI/CD 工作流不存在
- ADR 架构决策记录文档不存在

这些基础设施对项目的长期可维护性、发布流程和团队协作至关重要。

## 目标

1. **基准测试**：添加性能基准测试，确保关键路径性能达标
2. **模糊测试**：添加模糊测试，防止解析器和合并引擎的边角情况 panic
3. **CI/CD**：添加完整的 GitHub Actions 工作流
4. **ADR 文档**：创建架构决策记录，记录设计选择

## 非目标

- 不修改现有核心功能
- 不添加新的功能特性
- 不重构现有代码

## 范围

### 涉及文件

- `benches/` - 添加 4 个基准测试文件
- `fuzz/` - 添加模糊测试 crate 和 3 个 fuzz targets
- `.github/workflows/` - 添加 4 个工作流文件
- `docs/adr/` - 添加 40 个 ADR 文档

### 不涉及

- `src/` 核心代码修改
- `confers-macros/` 过程宏修改
- `confers-cli/` CLI 工具修改
- `tests/` 集成测试修改

## 成功标准

1. `cargo check --all-features` 编译通过
2. `cargo bench` 基准测试可运行
3. `cargo fuzz run` 模糊测试可执行
4. CI 工作流可通过 `gh run list` 查看
5. ADR 文档可通过 `ls docs/adr/` 查看

## 风险

| 风险 | 级别 | 缓解 |
|------|------|------|
| 基准测试代码影响编译时间 | 低 | 使用 `#[bench]` 属性 |
| 模糊测试消耗资源 | 低 | CI 中限制运行时间 |
| ADR 文档内容不完整 | 中 | 优先记录关键决策 |
